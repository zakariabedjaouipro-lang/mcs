# نظام الموافقة على المستخدمين الجدد
# User Approval System

## نظرة عامة | Overview

تم تفعيل نظام الموافقة على المستخدمين الجدد. عندما يسجل أي مستخدم لدور يتطلب موافقة (مثل الطبيب أو مدير العيادة أو المسؤول)، يتم:

**An approval system has been implemented. When a user registers for a role requiring approval (doctor, clinic admin, etc.), the following happens:**

1. ✅ **إنشاء حساب Supabase** - يتم إنشاء حساب مستخدم جديد على الفور
2. ✅ **حفظ طلب الموافقة** - يتم حفظ بيانات الطلب في جدول `registration_requests`
3. ✅ **إرسال إشعار للمسؤولين** - يتم إرسال بريد إلكتروني لجميع السوبر أدمن بطلب الموافقة
4. ✅ **عرض الطلب في لوحة التحكم** - يظهر الطلب في صفحة "طلبات الموافقة المعلقة" للسوبر أدمن
5. ✅ **الموافقة أو الرفض** - يمكن للسوبر أدمن الموافقة أو رفض الطلب مع إرسال بريد للمستخدم

---

## التدفق الكامل | Complete Flow

### 1. مرحلة التسجيل | Registration Phase

```
المستخدم الجديد
    ↓
اختيار دور يتطلب موافقة (مثلاً طبيب)
    ↓
ملء بيانات التسجيل والضغط على "تسجيل"
    ↓
إرسال البيانات إلى advanced_auth_bloc
```

### 2. معالجة التسجيل | Registration Processing

```
advanced_auth_bloc
    ↓
التحقق من متطلبات الدور (requires_approval)
    ↓
إنشاء حساب Supabase بواسطة authService.signUpWithEmail()
    ↓
إنشاء طلب موافقة في جدول registration_requests
    ↓
إرسال إشعارات بريدية للسوبر أدمن
    ↓
إرسال حالة النجاح للشاشة
```

### 3. مرحلة الموافقة | Approval Phase

```
السوبر أدمن يفتح صفحة "طلبات الموافقة المعلقة"
    ↓
عرض قائمة بجميع الطلبات المعلقة
    ↓
السوبر أدمن يختار: موافقة ✓ أو رفض ✗
    ↓
إرسال الإجابة إلى advanced_auth_bloc
    ↓
تحديث حالة الطلب في قاعدة البيانات
    ↓
إرسال بريد تأكيد للمستخدم
```

## كيفية الاستخدام | How to Use

### للمستخدمين العاديين | For Regular Users

1. افتح تطبيق MCS
2. اضغط "تسجيل جديد"
3. اختر دوراً من الأدوار (مثلاً: طبيب)
4. ملء البيانات المطلوبة
5. اضغط "تسجيل"
6. سيظهر رسالة: "Registration submitted. Awaiting admin approval."
7. انتظر حتى يوافق المسؤول على طلبك
8. ستستقبل بريداً إلكترونياً بالموافقة أو الرفض

### للسوبر أدمن | For Super Admins

1. افتح لوحة التحكم (Admin Dashboard)
2. اذهب إلى "Pending Approvals" (طلبات الموافقة المعلقة)
3. ستظهر قائمة بجميع الطلبات المعلقة
4. اضغط على أي طلب لرؤية التفاصيل
5. اختر: **"✓ Approve"** (موافقة) أو **"✗ Reject"** (رفض)
6. إذا اخترت الرفض، اكتب سبب الرفض
7. سيتم تحديث الحالة فوراً وإرسال بريد للمستخدم

## الأدوار التي تتطلب موافقة | Roles Requiring Approval

أدوار معينة تتطلب موافقة السوبر أدمن قبل تفعيل الحساب:

- 🏥 **Doctor** (طبيب)
- 🏢 **Clinic Admin** (مدير العيادة)
- 👨‍💼 **Admin** (إداري)

أدوار أخرى لا تتطلب موافقة:
- 👤 **Patient** (مريض)
- 👨‍💼 **Receptionist** (موظف استقبال)

## قاعدة البيانات | Database

### جدول registration_requests
```sql
CREATE TABLE registration_requests (
  id UUID PRIMARY KEY,
  user_id UUID NOT NULL,              -- معرّف المستخدم
  role_id UUID NOT NULL,              -- معرّف الدور
  status VARCHAR DEFAULT 'pending',   -- pending, approved, rejected, under_review
  requested_data JSONB,               -- بيانات إضافية
  reviewed_by UUID,                   -- معرّف من وافق/رفض
  reviewed_at TIMESTAMP,              -- وقت المراجعة
  rejection_reason TEXT,              -- سبب الرفض
  created_at TIMESTAMP DEFAULT NOW()
);
```

### حالات الطلب | Request Status
- `pending` - قيد الانتظار / Awaiting review
- `approved` - تمت الموافقة / Approved
- `rejected` - تم الرفض / Rejected
- `under_review` - قيد المراجعة / Under review

## الرسائل البريدية | Email Notifications

### 1️⃣ إشعار للسوبر أدمن
متى يرسل؟ عند تسجيل مستخدم جديد لدور يتطلب موافقة
المحتوى:
- بيانات المستخدم (الاسم، البريد، الدور)
- رابط لعرض الطلب في لوحة التحكم
- طلب الموافقة السريعة

### 2️⃣ تأكيد الموافقة للمستخدم
متى يرسل؟ عند موافقة السوبر أدمن على الطلب
المحتوى:
- رسالة ترحيب بالموافقة
- الدور الموافق عليه
- رابط للدخول إلى الحساب

### 3️⃣ رسالة الرفض للمستخدم
متى يرسل؟ عند رفض السوبر أدمن للطلب
المحتوى:
- إخطار بالرفض
- سبب الرفض (حسب ما أدخله المسؤول)
- معلومات التواصل للدعم الفني

## معالجة الأخطاء | Error Handling

### في حالة فشل التسجيل
- يتم عرض رسالة خطأ "Registration failed: {سبب الخطأ}"
- لا يتم إنشاء حساب
- لا يتم حفظ طلب موافقة

### في حالة فشل إرسال الإشعار
- لا يؤثر على نجاح التسجيل
- يتم تسجيل الخطأ في السجلات
- المستخدم قد لا يستقبل البريد (لكن الموافقة تعمل)

### في حالة فشل أرسال بريد الموافقة/الرفض
- تتم الموافقة أو الرفض في قاعدة البيانات
- يتم تسجيل الخطأ
- قد لا يستقبل المستخدم البريد

## اختبار النظام | Testing the System

### مثال 1: تسجيل طبيب جديد
```
1. افتح التطبيق → تسجيل جديد
2. اختر: Doctor 🏥
3. أدخل البيانات
4. اضغط: Register
5. فتح البريد → اعثر على رسالة من "Pending Approval"
6. كطبير أدمن، افتح Dashboard → Pending Approvals
7. اضغط: Approve ✓
8. تحقق من البريد للمستخدم: رسالة موافقة
```

### مثال 2: رفض طلب تسجيل
```
1. عندما تظهر الطلبات المعلقة
2. اختر طلب
3. اضغط: Reject ✗
4. اكتب السبب: "Credentials not verified"
5. اضغط: Reject
6. المستخدم يستقبل بريد الرفض
```
