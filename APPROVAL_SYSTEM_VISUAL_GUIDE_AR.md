## 🎯 من يستخدم نظام الموافقة؟

### الأدوار التي تتطلب موافقة:
```
✅ الأدوار الآتية تتطلب موافقة من المسؤول قبل التفعيل:
├── 1️⃣ الطبيب (Doctor) - يجب موافقة المسؤول
├── 2️⃣ مدير العيادة (Clinic Admin) - يجب موافقة المسؤول
├── 3️⃣ المسؤول العام (Admin) - يجب موافقة المسؤول
└── 4️⃣ المسؤول الأعلى (Super Admin) - يجب موافقة المسؤول الأعلى

❌ الأدوار التي لا تتطلب موافقة (تفعيل فوري):
├── المريض (Patient) - تفعيل مباشر
└── موظف الاستقبال (Receptionist) - تفعيل مباشر
```

## 🔌 كيفية التكامل مع التطبيق

### 1️⃣ في ملف التوجيه (Router)

```dart
// أضف المسار التالي إلى GoRouter
GoRoute(
  path: '/admin/approvals',
  name: 'pending-approvals',
  pageBuilder: (context, state) => MaterialPage(
    child: const PendingApprovalsScreen(),
  ),
  // فقط للمسؤولين والمسؤولين الأعلى
  redirect: (context, state) {
    final userRole = // احصل على دور المستخدم
    if (userRole != 'super_admin' && userRole != 'admin') {
      return '/dashboard'; // أعد التوجيه إذا لم يكن مسؤول
    }
    return null;
  },
),
```

### 2️⃣ في لوحة التحكم (Admin Dashboard)

```dart
// أضف زر في قائمة المسؤول
ListTile(
  leading: Icon(Icons.approval),
  title: Text(isArabic ? 'طلبات الموافقة' : 'Approval Requests'),
  trailing: Badge(
    label: Text('$pendingCount'),
  ),
  onTap: () {
    context.go('/admin/approvals');
  },
)
```

### 3️⃣ في شاشة التسجيل

```dart
// بعد التسجيل الناجح، تحقق من حالة الموافقة
if (state is RoleBasedRegistrationSuccess) {
  if (state.pendingApproval) {
    // اعرض رسالة: "تم التسجيل بنجاح، في انتظار الموافقة"
    ScaffoldMessenger.of(context).showSnackBar(
      SnackBar(
        content: Text(
          isArabic
              ? 'تم تسجيل البيانات، في انتظار موافقة المسؤول'
              : 'Your request is pending admin approval'
        ),
      ),
    );
  } else {
    // يمكن الدخول مباشرة
    context.go('/dashboard');
  }
}
```

## 🎯 حالات الاستخدام

### الحالة 1️⃣: تسجيل طبيب جديد

```
1. الطبيب يدخل البريد والكلمة المرور
2. يختار دوره: "طبيب"
3. النظام ينشئ طلب موافقة
4. إرسال بريد للمسؤول
5. المسؤول يفتح لوحة الموافقات
6. يضغط "موافقة"
7. البريد يرسل للطبيب بتأكيد النجاح
8. الطبيب يستطيع تسجيل الدخول الآن
```

### الحالة 2️⃣: رفض تسجيل مريض (بدون موافقة)

```
1. المريض يسجل البيانات
2. يختار دوره: "مريض" (لا يحتاج موافقة)
3. الحساب يتفعل فوراً
4. يستطيع الدخول مباشرة
```

### الحالة 3️⃣: رفض طلب موظف

```
1. الموظف يسجل البيانات العيادة
2. يختار دور استقبالي (يحتاج موافقة)
3. البريد يرسل للمسؤول
4. المسؤول يفتح لوحة الموافقات
5. يضغط "رفض"
6. يدخل السبب: "البيانات غير صحيحة"
7. يرسل الرفض
8. البريد يصل للموظف بتفسير السبب
```
